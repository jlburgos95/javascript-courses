📁 El archivo de configuración TypeScript

Apuntes

El archivo de configuración tsconfig.json

  • Especifica la raíz de un proyecto
  • Permite configurar opciones para el compilador
// Para iniciar el proyecto y crear el archivo de configuración
tsc --init

El archivo de configuración tiene las siguientes propiedades, las cuales nos deja personalizar nuestro compilador de TS

{
	// Heredar de otro archcivo
	"extends": "./configs/base"
	"compilerOptions":{
		// Especificar la version que se va a generar como resultado de compilacion
		"target": "es5",
		// La configuracion de generación de modules
		"module": "commonjs",
		// Habilitar el modo estricto
		"strict": true,
		// Para eliminar comentarios del codigo fuente
		"removeComments": true,
		// Para especificar una carpeta de salida
		"outDir": "./dist",
	},
	"include": [
		"src/**/*.ts"
	],
	// Excluir archivos para la compilación
	"exclude": [
		"node_modules",
		"**/*.test.ts"
	]
}
tsc -> busca la configuración
tsc --project platzi -> Directorio que contiene la configuración
tsc file.ts -> omite la configuración

Cabe recalcar que por defecto la entrada de todo nuestros archivos serán la carpeta src

🎲 Tipado en TypeScript

Ideas/conceptos claves

Explicita.- Definir el tipo de dato al momento de crear la variable

Inferido.- TypeScript deduce el tipo de la variable a partir del valor de la variable

Apuntes

Dentro del lenguaje se pueden definir dos formas de tipos:

Explicita

nombreVariable: TipoDeDato

Inferido

nombreVariable = valorIncial

Tipos básicos

Tipos para datos simples en TypeScript, se tienen tipos primitivos

📊 Number, Boolean y String

Ideas/conceptos claves

String.- Cadena de texto

Apuntes

Tipo: Number

  • Valores numéricos

desde ECMA15

  • Valores hexadedcimales ⇒ 0xf00d;
  • Valores binarios ⇒ 0b01010
  • Valores octales ⇒ 0o742;

Tipo: Boolean

  • El tipo de dato más simple en TS
  • Dos únicos valores: true y false

Tipo: String

  • El tipo de dato para trabajar con datos textuales o cadenas
  • Así como en JS se pueden usar comillas dobles ("") y simples ('')

Tipo: String Template Strings

  • Permiten definir múltiples líneas de texto
  • Pueden contener expresiones o variables embebidas
  • Se debe usar el carácter back tick / back quote (`) y para expresiones ${expr}

🤷‍♂️ Any

Apuntes

  • Usando para capturar valores dinámicos
  • Los valores pueden cambiar de tipo en el tiempo:
    • APIs externas
    • Librerías de terceros
  • Usar any solo cuando no se sabe exactamente cual sera el valor que estará en la variable.

💭 Void, Never

Ideas/conceptos claves

Void.- Nos sirve mayormente en funciones

Apuntes

Void

  • Es lo opuesto de any
  • Representa la ausencia de tipo
  • Comúnmente se usa como tipo de retorno en funciones
  • los tipos null y undefined son un subtipo de void

Never

El tipo never se usa para representar valores que nunca se supone que ocurran. Por ejemplo, puede asignar never como el tipo de devolución de una función que nunca regresa. Esto puede suceder cuando una función siempre arroja un error o cuando está atrapada en un ciclo infinito.

  • Representa el tipo de valor que nunca ocurre
    • Funciones que lanzan excepciones
    • Funciones que nunca retornar un valor

🙅‍♂️ Null y Undefined

Ideas/conceptos claves

Si se declara la variable con tipado inferido de null o undefined, la variable permanece en el tipo any

Apuntes

  • TS ambos "valores" tienen sus respectivos tipos:
    • null
    • undefined

Null y Undefined como subtipos

  • Null y Undefined se pueden asumir como subtipos de los otros tipos de datos
  • Significa que se puede asignar null y undefined a una variable de tipo string, por ejemplo

Null y Undefined la opción —strictNullChecks

  • Solo permite asignar null y undefined a una variable de tipo any o sus tipos respectivos
  • Ayuda a evitar errores comunes en programación de apps en el ámbito JS.

Esta configuración nos ayuda a prevenir posibles errores a futuro

Configuración en consola:

tsc file.ts --strictNullChecks

Configuración en tsconfig.ts

{
	"strict": true
	"strictNullChecks": true
}

🥽 Aserciones de Tipo

Ideas/conceptos claves

Apuntes

  • Cuando el programador puede conocer más que Typescript sobre el valor de una variable
  • Es un mensaje al compilador: "Confía en mí, sé lo que hago"
  • Se parece al casting de tipos en otros lenguajes de programación
  • Usa dos sintaxis: < Angle Bracket > y (variable as tipo)
// tenemos una cadena, TS confia en mi!
let message: string =
  (<string>username).length > 5
    ? `Welcome ${username}`
    : 'Username is too short';
// Sintaxis "as"
message =
  (username as string).length > 5
    ? `Welcome ${username}`
    : 'Username is too short';

🧬 Interfaces

Ideas/conceptos claves

Interfaces.- Modelos tipados de objetos

Apuntes

Constituyen una forma poderosa de definir "Contratos" tanto para tu proyecto, como para el código externos del mismo

Propiedades Opcionales

  • No todas las propiedades de una interfaz podrían ser requeridas
  • Establecemos una propiedad como opcional con el símbolo (?) después del nombre.

Propiedades de solo lectura

  • Algunas propiedades de la interfaz podrían no ser modificables una vez creado el objeto.
  • Esto es posible usando readonly antes del nombre de propiedad
// Interfaz: Usuario
interface User {
  readonly id: number; // solo lectura
  username: string;
  isPro: boolean;
}

Extendiendo Interfaces

Las interfaces pueden extenderse unas de otras. Esto permite copiar los miembros ya definidos en una interfaz a otra, ganando flexibilidad y reusabilidad de componentes

enum PhotoOrientation {
  Landscape,
  Portrait,
  Square,
  Panorama
}

interface Entity {
  id: number;
  title: string;

}

interface Album extends Entity {
  // copia de los atributos de Entity
  description: string;
}

interface Picture extends Entity {
  orientation: PhotoOrientation;
}

🏛️ Clases

Ideas/conceptos claves

Apuntes

Clases y Constructores

A partir de ECMAScript 2015 es posible construir class y hacer uso del paradigma de la Programación Orientada a Objetos en JavaScript.

TypeScript permite aplicar estas técnicas sin tener que esperar por otra versión.

Miembros Públicos

TypeScript define un modificador de acceso público por defecto para los miembros de clase.

También es posible marcar un miembro como público usando la palabra reservada public

Miembros Privados

TS define una manera propia de declarar o marcar un miembro como privado usando la palabra reservada private

ECMAScript

TypeScript también soporta (a partir de la versión 3.8) la nueva sintaxis JS para miembros privados: #atributo

Esta característica puede ofrecer mejores garantías de aislamiento en miembros privado

Métodos Set y Get

TS soporto los métodos accesores set y get como una forma de interceptar los accesos a los miembros privados de un objeto

get pictures() {
    return this._pictures;
  }

  set pictures(pictures: Picture[]) {
    this._pictures  = pictures;
  }

Herencia de Clases y Miembros Protegidos

TS soporta este patrón común en el mundo de la POO. Implementa la habilidad de extender código de clases existentes a través de la herencia. Con la palabra reservada extends.

la palabra reservada protected hace que las propiedades de una super clase no este publica ni privada, solo estará en el ámbito de la clase que la heredara.

Clases Abstractas

son base de donde otras classes podrían derivarse. A diferencia de una interfaz, una clase abstracta puede implementar funciones para sus instancias. Con la palabra reservada abstract

Propiedades Estáticas y Propiedades de solo lectura

Las clases por lo general definen atributos y métodos aplicables a las instancias de las mismas.

A través de la palabra reservada static se puede definir un miembro visible a nivel de clase.

Al igual que las interfaces, podemos usar la palabra reservada readonly ara marcar el miembro de una clase como solo lectura.
